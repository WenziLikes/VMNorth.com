# Demo Script

Use this as a short walkthrough for recruiters, hiring managers, or interviewers.

## 2-Minute Version

VMNorth.com is a production-oriented digital studio platform. It includes a multilingual public website, portfolio project pages, pricing and service pages, client brief intake, visitor chat, and a secure admin workspace with operations views.

The strongest engineering signals are:

- Full-stack ownership across frontend, backend, auth, realtime behavior, and deployment
- Product thinking beyond a landing page
- Secure admin design with passkeys, TOTP, recovery codes, and audit logging
- Admin operations for saved briefs, local analytics, content copy, release CI, health, and retention
- Production readiness through Docker, environment checks, release checks, and smoke validation
- Documentation written for both engineering handoff and non-technical review

## Suggested Walkthrough

1. Open `https://vmnorth.com/` and show the brand, navigation, localization, and responsive UI.
2. Open `https://vmnorth.com/services/overview` and `https://vmnorth.com/pricing` to show structured marketing and commercial routes.
3. Open `https://vmnorth.com/work/brand-websites` to show a dynamic portfolio project route backed by editable project content.
4. Open `https://vmnorth.com/brief` and explain how it turns a lead into a structured project intake.
5. Open `https://vmnorth.com/apps/flipclock-display` and mention the app-specific support/privacy release pages.
6. Show the admin sign-in screenshot and explain the passkey-first auth model with TOTP and recovery-code fallback.
7. Finish with the architecture overview and release process documentation.

## Interview Framing

Strong short summary:

Built a multilingual React and Node.js product platform with visitor chat, lead intake, secure admin operations, local analytics, content copy management, portfolio publishing, Dockerized deployment, CI checks, and production runbooks.

Good follow-up points:

- I treated the project as an operational product, not just a static website.
- I implemented custom chat and admin flows to show backend ownership.
- I documented the system so another engineer or reviewer could understand the product quickly.
- I designed the release process around repeatable verification rather than manual confidence.
